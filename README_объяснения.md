### 9. Uber Pickups in New York City/NewYork.ipynb
**Цель:** Кластеризация геолокаций поездок Uber

**Что делает каждый блок:**

```python
# Блок 1: DBSCAN кластеризация
```
- **Загружает** 100,000 записей о поездках Uber в Нью-Йорке
- **Преобразует** координаты (Lat, Lon) из градусов в радианы
- **kms_per_radian = 6371.0088** — радиус Земли в км, нужен для расчёта расстояний
- **epsilon = 0.2 / kms_per_radian** — радиус окрестности 0.2 км для DBSCAN
- **DBSCAN** с `metric='haversine'` — использует формулу гаверсинуса для расчёта расстояний на сфере
- **min_samples=20** — минимальное количество точек для образования кластера
- **Зачем:** DBSCAN находит плотные области точек и помечает выбросы как "шум". Идеально для геоданных, так как не требует заранее знать количество кластеров

```python
# Блок 2: HDBSCAN кластеризация
```
- **HDBSCAN** — улучшенная версия DBSCAN
- **min_cluster_size=50** — минимальный размер кластера
- **prediction_data=True** — позволяет предсказывать кластеры для новых данных
- **outlier_scores_** — оценка аномальности каждой точки
- **Зачем:** HDBSCAN автоматически определяет количество кластеров и лучше работает с кластерами разной плотности

## 🔑 КЛЮЧЕВЫЕ КОНЦЕПТЫ

### 1. Масштабирование признаков (StandardScaler)
**Зачем нужно:**
- KMeans, PCA и многие другие алгоритмы используют расстояния между точками
- Если признаки в разных масштабах, модель будет отдавать предпочтение признакам с большими значениями
- StandardScaler приводит все признаки к среднему=0, стандартному отклонению=1

**Пример:**
- Зарплата: 30,000-100,000
- Возраст: 18-80
- Без масштабирования зарплата будет доминировать

### 2. PCA (Principal Component Analysis)
**Зачем нужно:**
- Уменьшение размерности (например, 17 → 2 признака)
- Удаление коррелированных признаков
- Визуализация многомерных данных
- Ускорение вычислений

**Как работает:**
- Находит новые оси (главные компоненты), которые объясняют максимальную вариативность данных
- Первая компонента объясняет больше всего вариации, вторая — следующую по величине, и т.д.

### 3. Метод локтя (Elbow Method)
**Зачем нужно:**
- Помогает выбрать оптимальное количество кластеров (k)
- Строит график WCSS vs количество кластеров
- "Локоть" — точка, после которой добавление кластеров даёт малый прирост качества

### 4. Дисбаланс классов
**Проблема:**
- Когда один класс намного многочисленнее другого (например, 99.83% vs 0.17%)
- Модель может просто всегда предсказывать большинство класс и иметь высокую точность

**Решения:**
- `class_weight='balanced'` — автоматическое взвешивание
- Изменение порога классификации
- SMOTE — синтетическое создание примеров миноритарного класса

### 5. DBSCAN и HDBSCAN
**Зачем нужно:**
- Не требуют заранее знать количество кластеров
- Находят кластеры произвольной формы
- Обнаруживают выбросы (шум)
- DBSCAN с `metric='haversine'` идеален для геоданных
