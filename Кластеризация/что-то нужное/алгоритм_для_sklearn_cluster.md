Чтобы уметь обучить **любую** модель кластеризации из `sklearn.cluster`, тебе нужно освоить универсальный алгоритм действий. В библиотеке `scikit-learn` все модели подчиняются единому интерфейсу, поэтому, выучив схему один раз, ты сможешь использовать любой алгоритм.

Вот полный чеклист знаний:

---

### 1. Единый интерфейс (API)
Все модели в `sklearn.cluster` имеют три главных метода:
*   `.fit(X)` — обучить модель (найти центры, плотности или связи).
*   `.fit_predict(X)` — обучить и сразу вернуть массив с номерами кластеров.
*   `.labels_` — атрибут, в котором хранятся результаты после обучения (те же метки).

> **Нюанс:** Не все модели имеют метод `.predict()` для новых данных (например, `DBSCAN` или `SpectralClustering` не умеют предсказывать для новых точек без переобучения, а `KMeans` — умеет).

---

### 2. Обязательная предобработка данных
Кластеризация почти всегда основана на **расстоянии**. Если один признак — «Возраст» (0–100), а другой — «Зарплата» (0–1 000 000), модель будет смотреть только на зарплату.

*   **Масштабирование:** Всегда используй `StandardScaler` или `MinMaxScaler`.
*   **Пропуски:** Кластеризация не работает с `NaN`. Их нужно заполнить (`df.fillna()`) или удалить.
*   **Категориальные данные:** Модели работают только с числами. Переводи текст в числа через `OneHotEncoder` или `LabelEncoder`.

---

### 3. Знание "Тройки Игроков" (какой алгоритм выбрать)
Тебе не нужно знать все 10+ алгоритмов, достаточно понимать эти три типа:

1.  **Центроидные (KMeans, MiniBatchKMeans):**
    *   *Когда:* Кластеры похожи на сферы, данных много, ты примерно знаешь количество групп.
2.  **Плотностные (DBSCAN, OPTICS):**
    *   *Когда:* Данные сложной формы (дуги, кольца), много шума (выбросов), ты **не знаешь** количество кластеров.
3.  **Иерархические (AgglomerativeClustering):**
    *   *Когда:* Тебе нужно понять структуру (кто во что входит) или данных мало (до 5–10 тысяч строк).

---

### 4. Как подбирать гиперпараметры
Для каждой модели есть свой "главный рычаг":

*   Для **KMeans**: Параметр `n_clusters`. Ищем через **Метод Локтя** (инерция) или **Коэффициент Силуэта**.
*   Для **DBSCAN**: Параметр `eps` (радиус поиска соседей). Ищем через метод **K-расстояния** (график расстояний до ближайшего соседа).
*   Для **Agglomerative**: Параметр `distance_threshold` или `n_clusters`.

---

### 5. Метрики качества (как понять, что получилось хорошо?)
Так как в кластеризации нет «правильных ответов» (целевой переменной $y$), мы используем внутренние метрики:

1.  **Silhouette Score (Коэффициент силуэта):** Показывает, насколько точка близка к своему кластеру и далека от чужого. (Диапазон: от -1 до 1. Выше — лучше).
2.  **Davies-Bouldin Index:** Измеряет среднее «сходство» между кластерами. (Ниже — лучше).
3.  **Calinski-Harabasz Index:** Отношение дисперсии между кластерами к дисперсии внутри кластера. (Выше — лучше).

```python
from sklearn.metrics import silhouette_score
score = silhouette_score(X, labels)
```

---

### 6. Снижение размерности (PCA)
Если у тебя 20 или 50 признаков, кластеризация может работать плохо из-за «проклятия размерности». 
*   Используй `from sklearn.decomposition import PCA`.
*   Сжимай данные до 2–3 компонентов перед кластеризацией. Это убивает шум и позволяет строить красивые графики.

---

### 7. Универсальный шаблон кода

Вот так выглядит обучение **любой** модели:

```python
from sklearn.cluster import KMeans, DBSCAN, AgglomerativeClustering
from sklearn.preprocessing import StandardScaler
from sklearn.metrics import silhouette_score

# 1. Подготовка
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# 2. Выбор модели (просто меняешь класс)
# model = KMeans(n_clusters=5)
# model = DBSCAN(eps=0.5, min_samples=5)
model = AgglomerativeClustering(n_clusters=5)

# 3. Обучение
labels = model.fit_predict(X_scaled)

# 4. Оценка
print(f"Silhouette Score: {silhouette_score(X_scaled, labels)}")

# 5. Визуализация (если 2 признака)
import matplotlib.pyplot as plt
plt.scatter(X_scaled[:, 0], X_scaled[:, 1], c=labels, cmap='viridis')
```
